Philosphy

All plugins follow very simple but strict design guidlines which help to keep the library lightweight and easy to use.

Data-Attribute API

All the plugins provided by Accelerate are designed so that they can run out of the box without writing a single line of code. This makes it easier to rapidly add common basic functionality to your website with minimum effort as the plugins broadcast their various events in a namespaced event system.

Whilst this is appropriate for most websites, sometimes developers need to tweak the initialisation code to their needs so each plugins default behaviour can be disabled by unbinding all events on the document body like this:


// Target and disable all plugins.
$(document.body).off(".accelerate");

// Target and disable a specific plugin.
$(document.body).off("carousel.accelerate");

// Listen for and act upon a plugins exposed event.
$(document.body).on("slid.carousel.accelerate", "[data-carousel]", function (){
       
    // Run an event on the slid event.
                         
});

Programatic API

All the plugins providede by Accelerate can also be used through the standard jQuery API.

All the plugins are chainable, returning the collection that has been acted upon. All the plugin methods also accept an options object containing options that override the plugin defaults, a string calling a specific method or can be initialised parameterless to run with default behaviour:


// Chain the plugin method.
$(".overlay").overlay().addClass("foo");

// Initiaze the method with the different patterns.
$(".overlay").overlay() // default
$(".overlay").overlay({crosshair:true}) // add crosshair focusing
$(".overlay").overlay("show") // initialize and invoke the show moethod.

show.accordion

shown.accordion

hide.accordion
This event is fired immediately when the plugins hide instance method is invoked.
hidden.accordion
This event is fired once the collapsed region of the given instance has been hidden from the user

Google Map accelerate.googlemap.js

About

The google maps plugin renders an instance of a map containing a single marker. The map is fully responsive and the marker is also customizable.

Default Layout

Custom marker icon

Markup


<!-- Launch a google map with a centre point -->
<div id="google-map" data-googlemap='{"center":{"lat":55.86290,"lng":-4.251755}}'>
        
</div>

Methods

.googlemap(options)
Initialises the plugin with an optional options object
.googlemap("show")
Shows the current google map instance.
.googlemap("resize")
Triggers the resize event of the map instance.

Events

The GoogleMap plugin exposes the folowing events that allow you to hook into its behaviour.

show.googlemap
This event is fired immediately when the plugins show instance method is invoked.
shown.googlemap
This event is fired once the collapsed region of the given instance has been made visible to the user.
resize.googlemap
This event is fired immediately when the plugins resize instance method is invoked.
resized.googlemap
This event is fired once the given instance has been resized.

Notes

Since the Google Map plugin is initialised on page load the code required to disable the built in start up functionality of the plugin is slightly different.


 
// UnBind the built in loading event.
$(window).unbind("load.googlemap.accelerate");

// UnBind the built in resize event.
$(window).unbind("resize.googlemap.accelerate");

// Extending the map functionality
$(document.body).on("shown.googlemap.accelerate", "#google-map", function (event) {
      
    // Get the map and it's default marker
    var data = $(this).data("map"),
        map = data.map,
        marker = data.markers[0],

        // Create an infowindow.
        infowindow = new google.maps.InfoWindow({
        content: "This is a test."
    });

    // Bind the click event.
    google.maps.event.addListener(marker, "click", function () {
        infowindow.open(map, marker);
    });

});
